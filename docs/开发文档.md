# SoloCrop 开发文档

## 技术栈
- 前端：TypeScript + Vite + Vue3 + Element Plus
- 状态管理：Pinia
- 本地存储：IndexedDB (Dexie) + LocalStorage
- 视频处理：FFmpeg.wasm（本地运行时）
- 测试：Vitest

## 目录结构
- `frontend/src/views`：页面（`/media`、`/editor/:projectId`、`/settings`）
- `frontend/src/stores`：`media/split/editor/timeline/subtitle`
- `frontend/src/components/editor`：时间轴和预览组件
- `frontend/src/features/split`：自动分段模块
- `frontend/src/features/editor`：导出、队列、封面
- `frontend/src/features/subtitle`：本地 ASR
- `frontend/src/features/template`：模板库与模板解析
- `frontend/src/db`：IndexedDB 封装
- `frontend/src/types`：公共类型

## 自动分段模块
1. `planner.ts`
- 负责按总时长和分段秒数生成目标切点。

2. `keyframeAlign.ts`
- 负责切点关键帧对齐（容差窗口 1 秒）。
- 无命中时跳到下一个关键帧。

3. `ffmpegSplitService.ts`
- 加载 FFmpeg.wasm。
- 快速模式：`-c copy`；精确模式：`libx264 + aac`。
- 初始化超时保护、错误码检查、本地运行时强制路径。

4. `download.ts`
- 批量下载和目录写入（File System Access API）。
- 目录失败自动回退浏览器下载。

## 时间轴与编辑器
1. `timelineStore.ts`
- 维护 `TimelineProject`（video/audio/subtitle/sticker 四轨）。
- 支持旧草稿 `clips -> timeline` 迁移。
- 支持片段拖拽、裁剪、分割、磁吸、冲突挪位。
- 提供撤销/重做（50 步）。
- 提供模板一键应用：批量更新视频滤镜/转场并追加模板贴纸、字幕。

2. `EditorView.vue`
- 新流程：`时间轴 -> 字幕 -> 导出`。
- 默认导出模式为 `final` 成片导出。
- 新增“滤镜与转场（首版）”面板：支持按片段调整亮度/对比度/饱和度与淡入淡出。
- 新增“贴纸素材库（首版）”面板：支持添加文本贴纸并编辑位置/缩放/透明度。
- 新增“模板系统（首版）”面板：内置模板可一键应用到当前时间轴。
- 增加编辑器初始化门控：`timeline/media` 未完成加载时禁用“加入时间轴/保存工程”，避免初始化覆盖用户操作。

3. 组件
- `PreviewPlayer.vue`：播放头与视频预览联动。
- `PreviewPlayer.vue`：支持贴纸叠加预览层。
- `TimelineRuler.vue`：刻度和播放头线。
- `TimelineTrack.vue` / `TimelineItem.vue`：轨道条目拖拽和裁剪。
- `SubtitleTrack.vue`：字幕段编辑（文本、时间、启用状态）。

## 自动字幕
1. `subtitleStore.ts`
- 管理字幕段、模型设置、识别进度。
- 支持字幕批量偏移与逐条编辑。

2. `features/subtitle/asrService.ts`
- 本地 ASR 模型档位：`tiny/base`。
- 首次使用时做本地模型缓存准备（键：`solocrop.subtitle.model.cache`）。
- 提供 `transcribeLocal` 接口，输出 `SubtitleSegment[]`。

## 导出链路
1. `features/editor/exportTimeline.ts`
- 旧链路：`exportTimelineClips`、`exportTimelineMerged`。
- 新链路：`exportFinalVideo({ timeline, subtitleSegments, ... })`。
- 成片导出流程：时间轴拼接 -> 可选字幕烧录 -> MP4 输出。
- 片段特效支持：`eq`（亮度/对比度/饱和度）与 `fade in/out`。
- 成片导出支持贴纸烧录（`drawtext` 叠加，按时间段启用）。
- 提供 `validateExportPlan`、`classifyExportError`、`toSrt`。

2. 导出队列
- 串行执行、取消、手动重试、自动重试（指数退避）。
- 错误分级（可重试/不可重试）。

3. `features/editor/exportQueueStorage.ts`
- LocalStorage 持久化导出队列。
- 持久化字段：`clips`、`pipelineVersion`、重试字段、错误分类字段。
- 刷新恢复时将 `pending/running` 自动改为 `cancelled`。

## 本地运行
```powershell
./scripts/start-all.ps1
```

## 测试
```powershell
cd frontend
npm run test
```

## 需求闭环流程
1. 新增完整功能先更新 `docs/需求文档.md` 对比矩阵（状态=`未开始`）。
2. 开发期间更新状态为 `进行中` 并绑定 `PlaywrightCaseID`。
3. 完成后执行：
- `npm run test`
- `npm run test:e2e`（或 `scripts/run-e2e.ps1`）
4. 结果为 PASS 后，将状态改为 `已验收`。
5. 脚本门禁：
- `scripts/check-doc-sync.ps1`
- `scripts/check-requirement-coverage.ps1`
