# SoloCrop 部署文档

## 部署形态
纯前端静态站点部署，不依赖 API 服务和数据库。

## 打包
```powershell
cd frontend
npm install
npm run build
```

构建产物在 `frontend/dist`。
`npm install` 后会自动执行 `sync:ffmpeg-core`，将 `@ffmpeg/core` 复制到 `public/ffmpeg`，运行时不依赖外网 CDN。

## 本地启动
```powershell
./scripts/start-all.ps1
```
启动脚本会先执行 `sync:ffmpeg-core`，确保以下本地文件存在：
- `public/ffmpeg/ffmpeg-core.js`
- `public/ffmpeg/ffmpeg-core.wasm`
- `public/ffmpeg/worker/worker.js`
- `public/ffmpeg/worker/const.js`
- `public/ffmpeg/worker/errors.js`
启动时会强制占用 `5173` 端口并清理旧进程，避免误连到旧版 Vite 进程。
运行时完全走本地文件，不依赖 CDN。

## Playwright 验收执行
首次执行（安装浏览器）：
```powershell
cd frontend
npx playwright install chromium
```

本地关键流验收：
```powershell
./scripts/run-e2e.ps1
```

说明：
- `Playwright` 验收必须基于最新构建产物，流程为 `npm run build` 后再执行 `preview/e2e`。
- 直接对旧 `dist` 执行 e2e 可能出现“代码已改但用例仍失败”的假阴性。

仅执行前端 e2e（已存在服务）：
```powershell
cd frontend
npm run test:e2e
```

HTML 报告路径：
- `frontend/e2e/reports/index.html`

当前关键流用例包含：
- `E2E-MEDIA-IMPORT-001`
- `E2E-TIMELINE-EDIT-001`
- `E2E-SUBTITLE-AUTO-001`
- `E2E-TEMPLATE-001`
- `E2E-FILTER-TRANSITION-001`
- `E2E-STICKER-001`
- `E2E-EXPORT-FINAL-001`
- `E2E-EXPORT-RETRY-001`
- `E2E-USABILITY-PATH-001`

## 部署到 IIS/Nginx
- 将 `frontend/dist` 全量发布到站点目录。
- 配置 SPA 回退到 `index.html`。

## 浏览器要求
- Chrome/Edge 最新版。
- 若需“直接写入指定目录”，需浏览器支持 File System Access API（Chrome/Edge）。
- 封面帧截图依赖浏览器本地视频解码与 Canvas，移动端弱性能设备处理高分辨率视频会更慢。
- 时间轴拖拽编辑首版按桌面交互优化，移动端仅保证浏览与导出可用。

## 性能建议
- 单次处理大文件时会占用较多内存。
- 建议先将长视频按更短时长分段，减少浏览器压力。
- 快速关键帧模式速度更快，但切点可能有轻微偏差。
- 合并导出单文件会先生成临时片段再拼接，耗时与内存占用高于逐片段导出。
- 高码率（如 8Mbps）会增加输出体积与编码耗时，移动端建议优先使用自动或 2Mbps。
- 当选择了不兼容导出组合（如“合并导出 + 源分辨率 + 多素材”）时，界面会弹出修复提示并可一键切换推荐预设。
- 可在导出过程中手动取消当前任务；取消后会中止本次 ffmpeg 执行并继续队列中后续任务。
- 导出队列历史存储在浏览器 LocalStorage，清理站点数据会导致历史记录丢失。
- 对失败任务支持快速重试；建议重试前检查输出目录权限与导出参数是否合理。
- 可开启自动重试并设置最大次数；网络/权限类错误通常不会因自动重试自愈，建议先修复根因。
- 系统会将失败原因分类为“可重试/不可重试”；不可重试错误（如权限/参数问题）默认不会触发自动重试。
- 自动字幕使用本地 ASR 流程，首次使用需要准备模型缓存：
  - `tiny` 约 78MB（推荐默认）
  - `base` 约 142MB（更慢、占用更高）
- 低内存设备建议使用 `tiny`，并将素材分段后再识别。
