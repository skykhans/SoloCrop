## DoD（Definition of Done）统一标准
1. 代码实现完成，核心路径可执行。
2. 文档同步更新：`功能文档.md`、`开发文档.md`、`部署文档.md`、`数据结构文档.md`、`需求文档.md`。
3. `npm run test` 通过。
4. 对应功能至少 1 条 Playwright 关键流通过（PASS）。
5. 错误提示明确，含可行动建议。

## 发布前清单
1. 所有“本轮必须项”状态必须为 `已验收`。
2. 不允许本轮必须项处于 `未开始/进行中/可用`。
3. 本轮必须项关联 PlaywrightCaseID 最近结果必须为 `PASS`。
4. `scripts/check-doc-sync.ps1` 与 `scripts/check-requirement-coverage.ps1` 都通过。

## 新增完整功能流程
1. 立项：在对比矩阵新增条目，状态=`未开始`。
2. 开发中：状态=`进行中`，补全 DoD 与 PlaywrightCaseID。
3. 开发完成：状态=`可用`，同步更新全部文档。
4. 自测验收：执行 `npm run test` + `npm run test:e2e`，更新结果。
5. 验收通过：状态=`已验收`；若 FAIL，继续修复直到 PASS。
